# Лабораторная работа 2. Разработка лексического анализатора (сканера)
## Цель работы:
Изучить назначение и принципы работы лексического анализатора в структуре компилятора. Спроектировать алгоритм (диаграмму состояний) и выполнить программную реализацию сканера для выделения лексем из входного текста. Интегрировать разработанный модуль в ранее созданный графический интерфейс языкового процессора.
## Студент: Осинцева Д.А. АП - 327  
## Постановка задачи

Разработать лексический анализатор (сканер) в соответствии с индивидуальным вариантом задания, интегрировать его в приложение из лабораторной работы №1 и обеспечить наглядный вывод результатов. 



Требования к реализации сканера:


  Спроектировать диаграмму состояний конечного автомата, реализующего сканер, согласно варианту задания.

  
  Разработать программный модуль лексического анализа, который принимает на вход строку (исходный текст программы);  

  
  выделяет все допустимые лексемы согласно варианту; классифицирует лексемы по типам;

  
  учитывает многострочность входного текста; 

  
  любые символы, не соответствующие ни одному из допустимых типов лексем, считает недопустимыми и выводит сообщение об ошибке с указанием позиции.

Требования к интеграции и интерфейсу:
    Встроить сканер в ранее разработанный интерфейс и связать его с кнопкой "Пуск" и соответствующим пунктом меню.
    Реализовать навигацию по ошибкам: при щелчке на сообщении об ошибке в таблице результатов курсор в области редактирования должен устанавливаться на позицию недопустимого символа.

## Перечень допустимых лексем: 
- **0** – Строковое содержимое 
- **1** – Идентификатор 
- **2** – struct (ключевое слово)
- **3** – public (ключевое слово)
- **4** – string (ключевое слово)
- **5** – int (ключевое слово)
- **6** – void (ключевое слово)
- **7** – this (ключевое слово)
- **8** – Целое число 
- **9** – Число с плавающей точкой 
- **10** – Начало выражения / Начало выражения интерполяции (`{`)
- **11** – Конец выражения / Конец выражения интерполяции (`}`)
- **12** – Конец оператора (`;`)
- **13** – Начало перечисления аргументов (`(`)
- **14** – Конец перечисления аргументов (`)`)
- **15** – Оператор присваивания (`=`)
- **16** – Синтаксический знак (`.`)
- **17** – Синтаксический знак (`,`)
- **18** – Символ интерполяции (`$`)
- **19** – Синтаксический знак (`:`)
- **20** – Синтаксический знак (`"`)
- **21** – Разделитель (пробел)
- **22** – Ошибка

### Описание работы автомата

#### Числа

- **Целое число:** Если встречается цифра, сканер собирает все последующие цифры. Если после них нет точки — формируется целое число.
- **Число с плавающей точкой:** Если после цифр идет точка и за ней снова цифры — собирается дробная часть, формируется число с плавающей точкой.

#### Идентификаторы и ключевые слова

- При встрече буквы сканер собирает все буквы подряд
- Полученное слово сверяется со списком ключевых слов: `struct`, `public`, `string`, `int`, `void`, `this`
- Если совпадает — формируется ключевое слово, если нет — идентификатор

#### Строки

- Открывающая кавычка `"` начинает накопление символов внутри строки
- Закрывающая кавычка `"` завершает формирование строкового содержимого

#### Интерполяция строк

- Символ `$` перед кавычкой обозначает начало интерполированной строки
- Внутри строки при встрече символа `{` сканер временно выходит из режима строки и обрабатывает код внутри фигурных скобок
- При встрече символа `}` сканер возвращается в режим обработки строки

#### Пробелы

- Обрабатываются только значащие пробелы

#### Обработка ошибок

- Любой другой символ считается ошибкой
